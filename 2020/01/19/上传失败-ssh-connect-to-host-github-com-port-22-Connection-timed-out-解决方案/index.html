上传失败 ssh:connect to host github.com port 22:Connection timed out 解决方案

2020-01-19

因ssh连接超时导致更新博客失败的解决方案

问题描述

hexo -d 后报错如下：

ssh: connect to host github.com port 22: Connection timed out
fatal: Could not read from remote repository.
Please make sure you have the correct access rights
and the repository exists.

解决方案

0.先检查SSH连接是否成功，输入

ssh -T git@github.com
若报错
ssh: connect to host github.com port 22: Connection timed out
则采取下列方法
1.打开存放ssh的目录

cd ~/.ssh
ls

查看是否存在 id_rsa id_rsa.pun known_hosts 三个文件
若存在，则

vim comfig #新建文件

Host github.com
User YourEmail@163.com #User 后围殴github的账号名称
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443

#按esc后输入
:wq  #保存并退出

如果不习惯用命令行，也可以再cd ~/.ssh后输入pwd查看路径，从我的电脑按照相关路径打开后新建并输入同样的内容并保存

2.再次执行 ssh -T git@github.com 命令
出现提示输入yes即可
